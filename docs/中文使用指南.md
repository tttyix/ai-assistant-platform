### 执行模式说明

#### 模式 1：`aira+cc`（协作模式）⭐ 推荐

**流程：**
```
1. Aira 分析需求（架构师）
   ↓
2. Claude Code 执行（开发者）
   ↓
3. Aira 审查代码（质量监督）
   ↓
4. Aira 总结回复
```

**适用场景：**
- 创建完整项目
- 复杂功能开发
- 需要质量保证的任务

**示例：**
```json
{
  "task": "帮我创建一个基于 FastAPI 的博客系统，包含用户认证、文章管理、评论功能",
  "mode": "aira+cc",
  "context": {
    "technologies": ["FastAPI", "PostgreSQL", "JWT"],
    "features": ["用户注册登录", "文章 CRUD", "评论系统", "标签分类"]
  }
}
```

---

#### 模式 2：`aira_only`（仅分析）

**流程：**
```
1. Aira 分析需求
   ↓
2. 生成分析报告
   ↓
3. 返回建议
```

**适用场景：**
- 代码审查
- 技术咨询
- 方案设计
- 问题诊断

**示例：**
```json
{
  "task": "帮我审查这段代码的质量问题和潜在风险：[粘贴代码]",
  "mode": "aira_only"
}
```

---

#### 模式 3：`cc_only`（仅执行）

**流程：**
```
1. 直接调用 Claude Code
   ↓
2. 执行任务
   ↓
3. 返回结果
```

**适用场景：**
- 简单任务
- 快速修改
- 文件操作

**示例：**
```json
{
  "task": "在项目目录创建一个 README.md 文件，内容是项目介绍",
  "mode": "cc_only"
}
```

## 💡 实际使用示例

### 示例 1：创建待办事项应用

**使用 Swagger UI：**

1. 访问 http://localhost:8000/docs
2. 找到 **⚡ 工作流协作**
3. 点击 **POST /api/v1/workflows/execute**
4. 点击 **"尝试一下"**
5. 填写内容：
   ```json
   {
     "task": "帮我创建一个待办事项管理系统，支持添加、删除、完成任务",
     "mode": "aira+cc"
   }
   ```
6. 点击 **"执行"**
7. 查看返回结果

## ⚠️ 常见问题

### Q1: 服务无法访问？

**检查服务状态：**
```powershell
Invoke-WebRequest -Uri "http://localhost:8000/health"
```

**重启服务：**
```powershell
cd E:\ai-assistant-platform\backend
..\venv\Scripts\uvicorn src.main:app --reload
```

---

### Q2: 任务执行超时？

**原因：**
- 复杂任务需要更长时间
- 默认超时 10 分钟

**解决方法：**
1. 等待任务完成（可能需要 5-15 分钟）
2. 或调整 `.env` 中的 `WORKFLOW_TIMEOUT`

---

### Q3: 找不到创建的文件？

**文件位置：**
```
E:\ai-assistant-platform\
```

CC 会在工作目录创建文件，可以查看该目录。

---

### Q4: API 文档不是中文？

**清除浏览器缓存后刷新页面**，或访问：
```
http://localhost:8000/docs?reload=true
```

## 🎯 最佳实践

### 1. 任务描述要具体

❌ 不好：
```json
{
  "task": "帮我写代码"
}
```

✅ 好：
```json
{
  "task": "帮我创建一个 Python 类，实现栈数据结构，支持 push、pop、peek 操作"
}
```

---

### 2. 提供上下文信息

❌ 不好：
```json
{
  "task": "创建一个项目"
}
```

✅ 好：
```json
{
  "task": "创建一个用户管理系统",
  "context": {
    "technologies": ["FastAPI", "PostgreSQL", "JWT"],
    "features": ["用户注册", "登录认证", "密码加密", "会话管理"]
  }
}
```

---

### 3. 选择合适的模式

- **复杂项目** → `aira+cc`
- **代码审查** → `aira_only`
- **简单修改** → `cc_only`
